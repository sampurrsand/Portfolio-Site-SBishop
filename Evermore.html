Evermore

The objective of this project was to design a responsive web app for an online wedding supplies store that adheres to a set of partial branding guidelines and key messaging. Over the course of one month, I created brand guidelines, made low-fidelity wireframes, mapped out a user flow diagram, gathered feedback on a mid-fidelity prototype, and designed the UI for mobile, tablet, and desktop screens.

View Case Study

User Flow

One of the key features I wanted to focus on was providing users an intuitive way to filter products they're looking for. To start, I designed an ideal user flow diagram for the below user story:

"As a customer, I want to have access to advanced filtering options, so that I don't have to browse through a lot of products to find what I'm looking for."

Branding

I developed the brand guidelines based on the key messaging provided, and then I designed the application according to these requirements. I wanted Evermore to feel romantic, elegant, and timeless, so I chose colors and typography that would reflect these qualities.

Iterate

There were two key areas I received a lot of feedback: the home screen and the login screen. These screens went through multiple iterations.

Conclusion

Working on Evermore showed me how branding guides design decisions at every step of the process to ensure a cohesive product. I found myself referencing the brand guidelines and key messaging no matter how big or small of a choice I wanted to make.
